Pair Programming in DBC

By Ben Morimoto and Armando Reed

What is pair programming?

Pair programming is the process of working with one other developer on a single project. One of the benefits of pair programming is that it helps reduce errors that might occur if one was working by oneself. Also, having another person to bounce ideas off of is advantageous because you might come up with an idea that is even better than what only either individual could have accomplished. It also helps keep each person in check when working, because you need to be able to justify your code as you collaborate.

Why is pair programming important to DBC?

It is important because pair programming is getting to be a more common practice in this industry. We might find ourself in a position in the future where we have to pair program, and it is good to be able to practice it now while we are also learning to program. Also, by doing this during DBC, it keeps us socializing with each other. We get to learn how other people work, and it is important to be able to adapt to someones work style to be productive if needed. Another important aspect of pair programming in DBC is getting the feedback from your pair. That way you can work on how you interact with other people and adjust where necessary.

Compare Driver and Navigator

The navigator takes the role that is looking at the map, thinking about the big picture, and where the project is going. They want to make sure the work is focused and does not stray from what is actually tying to be accomplished. The Driver is the one that is actually doing the writing the code, must be sure to be engaged with the navigator, and comment when appropriate. In either role, communication with each other is the key to be productive and successful.
