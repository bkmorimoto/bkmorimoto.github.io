When to Use Classes vs. IDs

1/22/15

I want to discuss best practices when deciding to used a class or id attribute when you want to format the html of you website. After some quick research, it is actually quite straightforward on when you should use one or the other.

Class: The advantage of defining a class is that it can be used across multiple elements when they all need the same formatting.

In this example, we have defined the class "address" because we want to format each line of the address the same way without affecting ALL div elements. Because the classes can be reused, this makes the CSS coding much easier and less repetitive. It would look something like this:

Now all three pieces are bolded, the same size, and the same color.

You can also have multiple classes per element if you need some kind of combined formatting on certain elements of you choice.

We have combined both the address class and the fancy class, so the name on our webpage with not only be bolded, larger, and red; it will be also be formatted by fancy!

ID: This is unique to the element that it is defined under and CANNOT be used for multiple elements. Here I have defined my social media div, and since this is very specific to how I would format my three images, I would not reuse the same formatting anywhere else on my website:

An element can have both a class and an ID defined, but the ID takes precedence if there are any conflicting features.


This element has the class "name" and the ID "ben." Looking at the CSS we can see that font-size is defined in both the class and the ID, but because ID has precedence, the actual font size displayed with be 30 pixels.
